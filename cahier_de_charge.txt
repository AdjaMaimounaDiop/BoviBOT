  1. CONTEXTE ET OBJECTIFS
Dans le cadre du cours d'integration de l'IA et des bases de donnees avancees (Licence 3), vous devez
concevoir et realiser une application web de gestion d'elevage bovin integrant :
- Un assistant LLM connecte a une base MySQL (Text-to-SQL + actions)
- Des elements PL/SQL avances : procedures stockees, fonctions, triggers et events
L'objectif  est  de  demontrer  votre  maitrise  des  bases  de  donnees  avancees  et  de  l'IA  appliquee  a  un
contexte metier reel : la gestion d'un troupeau bovin.
  2. CONTEXTE METIER
Un  eleveur  gere  son  troupeau  de  bovins  :  suivi  des  animaux,  controle  sanitaire,  pesees  regulieres,
gestion de la reproduction, alimentation et ventes. L'application doit permettre :
- D'interroger les donnees en langage naturel
- D'enregistrer des pesees et ventes via le chat (avec confirmation)
- De recevoir des alertes automatiques generees par les triggers et events MySQL
  3. BASE DE DONNEES MYSQL (8 tables)
 Table  Description  Champs cles
 races  Races bovines  nom, origine, poids_adulte_moyen_kg
 animaux  Individus du troupeau  numero_tag, race_id, sexe, date_naissance, statut, mere_id, pere_id
 pesees  Historique des pesees  animal_id, poids_kg, date_pesee, agent
 sante  Actes veterinaires  animal_id, type, date_acte, veterinaire, prochain_rdv
 reproduction  Suivi gestations / velages  mere_id, pere_id, date_velage_prevue, statut
 alimentation  Rations alimentaires  animal_id, type_aliment, quantite_kg, cout_unitaire_kg
 ventes  Ventes d animaux  animal_id, acheteur, prix_fcfa, date_vente
 alertes  Alertes systeme (auto+manuelle)  type, message, niveau, traitee
  4. ELEMENTS PL/SQL OBLIGATOIRES
4.1  Procedures stockees (OBLIGATOIRES)
 Procedure  Role  Paramètres
 sp_enregistrer_pesee  Enregistre pesee + MAJ poids_actuel + calcul GMQ + alerte si faible animal_id, poids_kg, date, agent
 sp_declarer_vente  Enregistre vente + change statut animal en "vendu" + verification statut animal_id, acheteur, tel, prix, poids, date
4.2  Fonctions (OBLIGATOIRES)
 Fonction  Retourne  Utilisation
 fn_age_en_mois(animal_id)  INT - age en mois  Dans toute requete impliquant l age
 fn_gmq(animal_id)  DECIMAL - gain moyen quotidien  Analyse de croissance du troupeau
4.3  Triggers (OBLIGATOIRES)
 Trigger  Evenement  Action
 trg_historique_statut  BEFORE UPDATE animaux  Log changement de statut dans historique_statut

  5. FONCTIONNALITES LLM ATTENDUES
5.1  Mode CONSULTATION (Text-to-SQL)
-  "Liste tous les animaux actifs avec leur age et GMQ" → utilise fn_age_en_mois() et fn_gmq()
-  "Quels animaux ont un GMQ inferieur a 0.3 kg/jour ?"
-  "Quelles femelles velent dans les 30 prochains jours ?"
-  "Quel est le cout total d alimentation ce mois-ci ?"
-  "Quels animaux n ont pas ete vaccines depuis 6 mois ?"
5.2  Mode ACTION (via procedures stockees)
-  "Enregistre une pesee de 325 kg pour TAG-001 aujourd hui" → CALL sp_enregistrer_pesee()
-  "Declare la vente de TAG-003 a Oumar Ba pour 280 000 FCFA" → CALL sp_declarer_vente()
-  Confirmation obligatoire avant tout appel de procedure
  6. EXEMPLES DE DIALOGUES LLM ATTENDUS
Consultation :
-  Utilisateur : "Quels animaux ont un GMQ inferieur a 0.3 kg/jour ?"
     ->  SQL  :  SELECT  a.numero_tag,  a.nom,  fn_gmq(a.id)  as  gmq  FROM  animaux  a  WHERE
a.statut="actif" HAVING gmq < 0.3
   -> Reponse : "2 animaux avec GMQ faible : TAG-006 (0.25), TAG-007 (0.18)"
Action avec confirmation :
-  Utilisateur : "Enregistre 320 kg pour TAG-001 aujourd hui"
   -> LLM identifie : animal_id=1, poids=320, date=2026-03-27
   -> Systeme : "Confirmer la pesee : TAG-001 / Baaba = 320 kg le 2026-03-27 ? (Oui/Non)"
   -> Apres oui : CALL sp_enregistrer_pesee(1, 320.0, "2026-03-27", "BoviBot")
   -> Reponse : "Pesee enregistree. GMQ actuel : 0.5 kg/jour. Aucune alerte."
